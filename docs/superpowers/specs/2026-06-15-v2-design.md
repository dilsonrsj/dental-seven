## 1. Objetivo da v2

Transformar o MVP demo (gate por senha única) em **produto multi-tenant real**: cada clínica com conta própria, auth Supabase, roles, módulos por plano, trial de 7 dias e integração Asaas (cobrança no 8º dia).

A v1 permanece funcional em dev com seed demo; o deploy público da v1 foi **adiado** — primeiro deploy será v2 ou posterior.

## 4. Modelo de assinatura

### Status da clínica

```sql
-- colunas em clinics (migration v2)
subscription_status text not null default 'trialing'
  -- trialing | active | past_due | expired | canceled
trial_ends_at timestamptz not null
plan_key text not null default 'essencial'
  -- essencial | conecta | inteligente | completo
asaas_customer_id text
asaas_subscription_id text
```

### Fluxo trial → cobrança

```
Cadastro (dia 0)
  → subscription_status = 'trialing'
  → trial_ends_at = now() + 7 days
  → Asaas: cria customer (sem cobrança imediata)
  → Asaas: cria subscription com firstDueDate = trial_ends_at

Dia 1–7
  → Acesso total ao plano escolhido (módulos habilitados)

Dia 8
  → Asaas tenta cobrança (PIX/boleto/cartão conforme cadastro posterior)
  → PAYMENT_CONFIRMED → 'active'
  → Falha → 'expired' ou 'past_due'

Trial expirado ('expired')
  → Login com e-mail/senha funciona
  → Middleware/layout detecta status → PaywallOverlay
  → Todas as actions bloqueadas (read-only mínimo ou zero funcional)
  → Banner: "Seu período de teste encerrou. Regularize para continuar."
  → CTA: "Assinar agora" → fluxo Asaas (cadastro pagamento)
  → E-mail automático no dia 8 (e opcional lembrete dia 6)
```

**Sem cartão no cadastro:** cliente informa forma de pagamento apenas ao converter (dia 8 ou antes, se quiser antecipar).

## 5. Paywall (trial expirado)

Componente `PaywallOverlay` no `(app)/layout.tsx`:

- Exibe quando `subscription_status IN ('expired', 'past_due')` e usuário não é `super_admin`
- Bloqueia interações (pointer-events no conteúdo principal)
- Permite: logout, link para assinatura, exportação de dados (LGPD — direito de portabilidade mesmo inadimplente)
- SuperAdmin DR7 não vê paywall

Mensagem automática (e-mail):

- Assunto: "Seu trial Dental Seven encerrou"
- Corpo: resumo do plano, link para pagamento, contato DR7

## 7. Auth e roles

```sql
profiles (
  id uuid references auth.users primary key,
  role text not null,          -- super_admin | clinic_admin | dentist
  clinic_id uuid references clinics,
  dentist_id uuid references dentists,
  full_name text,
  created_at timestamptz
)
```

| Papel | Escopo |
|-------|--------|
| `super_admin` | Todas as clínicas; `/admin` |
| `clinic_admin` | Sua clínica; configurações; billing |
| `dentist` | Agenda, pacientes (sem billing) |

Middleware: substituir cookie demo por sessão Supabase Auth.

RLS: políticas `clinic_id = auth.clinic_id()` (via JWT claim ou lookup em `profiles`).

## 8. Integração Asaas

### Endpoints (server-side)

- `POST /api/billing/create-customer` — onboarding
- `POST /api/billing/create-subscription` — após escolha de plano
- `POST /api/webhooks/asaas` — eventos de pagamento

### Webhooks relevantes

| Evento | Ação |
|--------|------|
| `PAYMENT_CONFIRMED` | `subscription_status = 'active'` |
| `PAYMENT_OVERDUE` | `subscription_status = 'past_due'` |
| `SUBSCRIPTION_DELETED` | `subscription_status = 'canceled'` |

Variáveis de ambiente: `ASAAS_API_KEY`, `ASAAS_WEBHOOK_TOKEN`, `ASAAS_ENV=sandbox|production`.
