---

## 1. Objetivo

Entregar o módulo **Fornecedores** para clínicas no plano **Completo**: cadastro de contatos de reposição, vínculo preferencial insumo→fornecedor e botão **Pedir reposição** via WhatsApp a partir de alertas de estoque — complemento operacional de Estoque e Financeiro.

## 2. Decisões

| # | Decisão |

### Fora do escopo

- Múltiplos fornecedores por insumo

- Dentista acionando reposição ou vendo fornecedores
- E-mail automático (`mailto:`) — fallback só exibição/cópia manual

## 4. Modelo de dados

```sql
-- v5.1: suppliers + preferred link on supplies

create table suppliers (
  id uuid primary key default gen_random_uuid(),
  clinic_id uuid not null references clinics(id) on delete cascade,
  name text not null,
  phone text,
  email text,
  notes text,
  is_active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

alter table supplies
  add column preferred_supplier_id uuid
    references suppliers(id) on delete set null;

create index idx_suppliers_clinic_active_name
  on suppliers(clinic_id, is_active, name);

create index idx_supplies_preferred_supplier
  on supplies(preferred_supplier_id)
  where preferred_supplier_id is not null;

alter table suppliers enable row level security;

create policy "suppliers_clinic" on suppliers for all
  using (clinic_id = public.current_clinic_id() or public.is_super_admin())
  with check (clinic_id = public.current_clinic_id() or public.is_super_admin());
```

**Regras de integridade:**

- `preferred_supplier_id` deve referenciar fornecedor da **mesma clínica** (validado na action)
- Ao desativar fornecedor: não remove vínculos; admin deve reatribuir se necessário
- Ao excluir fornecedor: `ON DELETE SET NULL` em `supplies.preferred_supplier_id`

## 5. Regras de negócio

### Permissões

| Ação | `clinic_admin` | `dentist` |
|------|----------------|-----------|
| Ver `/fornecedores` | ✅ | ❌ (redirect `/agenda`) |
| CRUD fornecedores | ✅ | ❌ |
| Vincular insumo→fornecedor | ✅ | ❌ |
| Botão **Pedir reposição** em `/estoque` | ✅ | ❌ |
| Ver lista de estoque (alertas) | ✅ | ✅ (leitura, sem botão) |

### WhatsApp — Pedir reposição

Pré-requisitos (todos):

1. Módulo `fornecedores` ativo
2. Insumo com alerta (`warning` ou `critical`)
3. `preferred_supplier_id` preenchido
4. Fornecedor com `phone` cadastrado

**Mensagem sugerida:**

```
Olá! Preciso repor o insumo {supply_name}. Saldo atual: {quantity_on_hand} {unit_label}. Mínimo: {min_quantity}.
```

**URL:** `https://wa.me/{phone_digits}?text={url_encoded_message}`

- Normalizar telefone: apenas dígitos; prefixar `55` se número local (10–11 dígitos)
- Abrir em nova aba (`target="_blank"`, `rel="noopener noreferrer"`)

**Sem telefone:**

- Botão desabilitado ou oculto
- Tooltip: *Cadastre telefone do fornecedor*
- Exibir e-mail do fornecedor (se houver) para cópia manual

### Dependências de módulos

| Módulo | Necessário para |
|--------|-----------------|
| `procedimentos` | Insumos (`supplies`) existentes |
| `estoque` | Alertas de mínimo e botão de reposição |
| `fornecedores` | CRUD e vínculos |
| `financeiro` | **Não** obrigatório |

## 7. UI

### `/fornecedores` — admin

- Título **Fornecedores**
- Tabela: nome, telefone, e-mail, status (ativo/inativo), ações (editar, desativar)
- Botão **Novo fornecedor** → modal (nome*, telefone, e-mail, observações, ativo)
- Seção **Insumos vinculados:** tabela de insumos ativos com select de fornecedor preferencial (somente fornecedores ativos)

### `/estoque` — admin (quando `fornecedores` ativo)

- Coluna ou ação **Pedir reposição** visível em linhas com alerta
- Ícone/link WhatsApp; desabilitado se sem fornecedor ou sem telefone

### Menu

- Ícone: `Truck` (lucide-react)
- Após **Financeiro**, antes de **WhatsApp**
