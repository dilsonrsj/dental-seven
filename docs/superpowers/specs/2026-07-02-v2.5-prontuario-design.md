## 1. Objetivo

Permitir que clínicas no plano **Completo** façam **upload de documentos** (PDF, JPG, PNG) na ficha do paciente — histórico importado de outros sistemas, laudos e exames — com armazenamento seguro por clínica no Supabase Storage.

---

## 2. Decisões

| # | Decisão |
|---|---------|
| 1 | Módulo `prontuario` — só visível se `clinic_modules` tiver `prontuario` enabled (plano Completo ou SuperAdmin) |
| 2 | Rota `/pacientes/[id]/prontuario` — aba na ficha do paciente |
| 3 | Bucket Supabase Storage: `patient-documents` (privado) |
| 4 | Path: `{clinic_id}/{patient_id}/{document_id}/{filename}` |
| 5 | Tabela `patient_documents` com metadados; arquivo no Storage |
| 6 | Upload via server action + signed upload URL (ou upload direto admin + RLS storage) |
| 7 | Formatos v2.5: `application/pdf`, `image/jpeg`, `image/png` — máx. **10 MB** por arquivo |
| 8 | Roles: `clinic_admin` e `dentist` podem listar e fazer upload; `super_admin` leitura/export |
| 9 | Paywall v2: upload bloqueado se assinatura inativa (mesmo padrão das outras actions) |
| 10 | Demo `DEMO_MOCK_DATA=true`: lista vazia + mensagem “disponível no plano Completo” sem Storage |

## 4. Modelo de dados

```sql
create table patient_documents (
  id uuid primary key default gen_random_uuid(),
  clinic_id uuid not null references clinics(id) on delete cascade,
  patient_id uuid not null references patients(id) on delete cascade,
  title text not null,
  mime_type text not null,
  storage_path text not null,
  file_size_bytes bigint not null,
  source text not null default 'imported'
    check (source in ('imported', 'generated', 'clinical')),
  uploaded_by uuid references profiles(id) on delete set null,
  created_at timestamptz not null default now()
);
```

Índices: `(clinic_id, patient_id)`, `(patient_id, created_at desc)`.

---

## 5. Storage policies

- SELECT/INSERT/DELETE: usuário autenticado com `current_clinic_id()` = primeiro segmento do path
- SuperAdmin: leitura global
- Path validado no server action antes de gravar metadados

---

## 6. UI

- Aba ou sub-rota na ficha: **Informações** (`/pacientes/[id]`) | **Prontuário** (`/pacientes/[id]/prontuario`)
- Estado vazio: “Nenhum documento. Arraste PDF ou imagem para importar histórico.”
- Card por documento: ícone por mime, título, data, botão Baixar
- Zona de upload com limite de tamanho e tipos aceitos visíveis
